# Lesson 03 – Making Changes

## Overview

When working with Claude Code, you will often modify existing projects.
This lesson covers techniques that help you guide Claude effectively when making changes to your code.

---

## Using Screenshots

Screenshots help Claude understand UI changes clearly.

To paste a screenshot into Claude:

Ctrl + V

This allows you to show exactly what part of the interface should be modified.

Example:

* Change button color
* Adjust layout
* Fix alignment issues

---

## Planning Mode

Planning Mode helps Claude analyze your project before making changes.

Enable Planning Mode:

Shift + Tab (twice)

Claude will then:

* Read more files in your project
* Explore the codebase
* Create an implementation plan
* Wait for your approval before coding

This is useful for complex tasks affecting multiple files.

---

## Thinking Modes

Claude supports different reasoning levels.

Think – basic reasoning
Think more – deeper reasoning
Think a lot – extensive reasoning
Think longer – longer reasoning time
Ultrathink – maximum reasoning power

These modes allow Claude to spend more time analyzing complex problems.

---

## Planning vs Thinking

Planning Mode is best for:

* Multi-file changes
* Large features
* Codebase exploration

Thinking Mode is best for:

* Debugging
* Complex logic
* Algorithms

You can combine both modes for complex development tasks.

---

## Key Idea

Use:

Planning Mode → for understanding the project
Thinking Mode → for solving difficult problems

Together they help Claude produce more accurate and reliable changes.
